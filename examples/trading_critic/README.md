# Trading-critic demo

A real, end-to-end EER demonstration: three iterations of a hand-authored
trading strategy critiqued by a TEE-attested LLM, producing a chain of three
verifying receipts that link by `previous_receipt_id` and share a stable
`input_commitment`.

This is the headline EER demo — it exercises every spec step that lands in
the v0.5 reference implementation (Steps 1, 2, 3, 4, 5 compose-hash subset, 6)
plus the chain pattern (§6.6) and input-commitment construction (§6.7).

## Status

Initial scaffolding. Helper scripts and `demo.py` land in subsequent commits
on this branch.

## Layout

- `bundle.json` — the evaluator bundle. `accepted_compose_hashes` is a
  placeholder; see `bundle_inspect.py` (added in a later commit) for the
  swap-in workflow.
- `strategies/v1.md` — committed; hand-authored MA-crossover strategy.
- `strategies/v2.md`, `v3.md` — gitignored; pre-generated by
  `author_strategies.py` (added later) and hand-edited.
- `receipts/{v1,v2,v3}.json` — gitignored; produced by `demo.py`.
- `.salt` — gitignored; the `SaltStore` for the demo's stable
  `input_commitment`.
